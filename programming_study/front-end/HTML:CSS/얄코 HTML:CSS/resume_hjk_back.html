    프로젝트 내용

    MAU 300명 규모의 Node.js기반 웹사이트로 월 100만원의 가치를 창출하고 있으며 현재 운영중에 있습니다. 이 사이트는 해냄학원의 학부모들에게 학생들의 다양한 성적 지표들을 손쉽게 보여주는 것을 목표로 만들어졌습니다. 사용된 기술 스택으로는 JavaScript, Node.js, Jest, MySQL 등을 사용했습니다. 프로젝트 구상 당시, 요청당 필요한 cpu 작업의 크기가 작다는 점에서 기술스택으로 Node.js를 선택했습니다. 초기에는 웹사이트에 문제가 생겨 업데이트를 해야할 때 어설픈 프로젝트의 구조, 테스트 코드의 부재로 업데이트마다 성적 공개가 몇시간 지연되는 등 큰 고난들을 겪었습니다. 그래서 이에 대한 필요성을 느껴 Jest를 이용해 테스트 코트를 작성하고, MVC패턴을 적용해 프로젝트의 구조를 개선하였습니다. 하지만 아직 UI 개선, 신 기능 추가, 보안성 개선 등 다양한 개선 사항이 있어 버젼을 지속적으로 업그레이드하면서 이를 달성하고자 하고 있습니다.

    프로젝트 내용

    여행을 할 때 큰 골칫거리인 무거운 짐에 대한 고민을 해결하고자 하는 목적에서 만든 프로젝트이며 여행객들의 짐을 배민 커넥트와 같이 일반인들이 옮길 수 있는 서비스를 제공하고자 했습니다. 제가 담당한 부분은 백엔드 API 개발, 구글 맵 API 활용, 프로젝트 배포이며 해당하는 기술스택으로는 Java, Spring, MySQL, AWS RDS 등을 사용하였습니다. 개인이 아닌 팀프로젝트였기 때문에 보다 많은 것을 경험하고 배울 수 있었습니다. 먼저, 코드부터 짜는 개발 방식이 아닌 프로젝트의 설계를 먼저 하였고 MVC 패턴 적용, 테스트 코드 작성을 통해 프로젝트의 구조가 견고해짐을 느꼈습니다. 이는 해냄학원을 개선하는데도 큰 영향을 미쳤습니다. 또한, 각자 개발한 것을 잘 합치는 것에 대한 고민을 하다 Git에 대한 필요성을 느껴, Git의 브랜치, 풀리퀘 등 협업을 위한 다양한 기능들을 익히고 활용하였습니다.

    자기소개

    자기소개

    1. 프로그래머가 되려는 이유 프로그래밍은 사람들의 상상을 현실로 만드는 간편하면서도 강력한 도구라고 생각합니다. 실제로 저는 웹사이트를 만들어 어머니의 퇴근시간을 한 시간 정도 앞당겼습니다. 이처럼 프로그래밍은 상상 속에서만 존재하던 무언가를 구현해 사람들의 삶에 큰 도움을 준다는 점이 제 마음을 움직였습니다. 저희 어머니는 학원에서 학생들의 성적을 학부모들에게 전송하는 업무를 담당하십니다. 그런데 성적을 학부모마다 개별 문자로 보내 많은 시간을 할애하고 있었습니다. 때문에 어머니가 이런 고민을 토로하셨을 때 저의 프로그래밍으로 어머니의 삶에 도움을 줄 수 있는 기회라 느꼈습니다. 그래서 어머니께 성적관리 웹사이트를 만드는 것을 제안했습니다. 이후 어머니의 다양한 요구사항들을 듣고 개발에 착수했습니다. 하지만 웹사이트 개발에 대한 지식이 많이 부족해 크게는 Node.js, Spring, Django 등 프레임워크의 선택부터 SQL Injection 방어, AWS를 통한 배포 등 다양한 문제들을 직면해 개발하는 동안 계속해서 자신의 한계를 돌파하는 느낌을 받았습니다. 결과적으로 만족스럽지 않지만, 요구사항에는 맞춘 웹사이트를 개발하여 배포하였고 어머니의 퇴근 시간을 한 시간 정도 앞당길 수 있었습니다. 이를 통해 프로그래머로서 많이 부족한 현재의 저도 누군가에게 도움이 되는 서비스를 만들 수 있다는 확신을 갖게 되었습니다. 또한, 새로운 것을 계속해서 배워나가는 점이 저의 흥미를 끌었습니다. 프로그래밍 지식들은 인터넷에 공식문서들이 정리가 잘 되어있어 내가 원한다면 얼마든지 깊이 들어갈 수 있고 커뮤니티 문화도 활성화되어 있어 질문하기도 용이합니다. 그렇기에 현재 상태에 만족하지 않고 계속해서 배워나가며 현재 저의 웹사이트가 가지고 있는 구조, 안정성 등을 보완해 사용자들의 편의를 개선하고 싶습니다. 또한 1인 개발을 하면서 넣고 싶은 기능들은 많았지만, 시간이나 규모적인 한계에 부딪힌 적이 많았습니다. 그래서 나중에는 뜻이 맞는 사람들과 혼자서는 불가능한 규모의 프로젝트들도 진행해보고 싶습니다. 2. 프로그래밍 학습 과정 개발자로서의 성장이 필요하다고 느낀 것은 완성된 프로젝트를 수정할 때였습니다. 이전까지는 단순히 과제와 목표에 급급해 아무런 기획 과정 없이 주먹구구식으로 프로젝트를 개발했습니다. 이런 프로젝트들은 추후에 단순한 기능을 바꿀 때도 코드 해석에 힘겨워할 뿐만 아니라 엉성한 프로그램 구조, 자료구조 때문에 문제가 생겨 어려움을 겪었습니다. 그래서 개발에 관한 지식들의 필요성을 느껴 책을 구매해 읽기 시작했습니다. 먼저, 수정할 당시 기존의 코드들을 다시 읽고 이해하기가 힘들어 가독성이 좋게 바꾸고자 “Clean Code”를 구매해 읽었습니다. 책에서는 어려운 개념들을 다루는 게 아닌 SOLID 원칙과 같이 학부 때 들었던 내용들을 잘 적용하는 방법을 다양한 예제들을 통해 알려줬습니다. 책을 통해 이론적으로 이해한 내용들을 실전에 적용하고자 알고리즘 코드나 프로젝트들을 개발할 때도 이를 염두에 두며 코드를 짜니 코드 해석에 드는 시간을 줄일 수 있었습니다. 또한, 람다식이나 foreach문 같이 비교적 최신 문법들을 몰라 코드를 읽기 힘들었던 적이 있어 자바에 대한 이해를 높이고자 “Effective Java”도 읽었습니다. 이를 통해 Optional, Defensive copy와 같이 기존에 몰랐던 개념들을 프로젝트에 적용해 안정성을 높였습니다. 이후에도 설계와 테스트 코드의 중요성을 인지하고 있었기에 프로그램 설계에 관한 책, 테스트 주도 개발에 관련된 책을 읽었고 현재는 컴퓨터 통신의 이해를 높이고자 네트워크 책을 읽고 있습니다. 이 외에도 알고리즘 해결 능력은 프로그램의 성능에 큰 영향을 미치기에 프로그래머의 기본 소양이라 생각했습니다. 그래서 군대를 가기 전 전역까지의 목표로 알고리즘 공부를 잡았습니다. 이를 실천하고자 군대에서 컴퓨터 사용이 불가능할 때는 책을 이용해 공부하였고 가능할 때는 알고리즘 문제 사이트에서 문제를 풀었습니다. 실제로 8개월간 휴가를 제외하고 매일 알고리즘 문제 푸는 데 성공하였고 전역 후 치룬 코딩테스트에서도 좋은 성과를 거두었습니다. 3. 몰입 경험 꾸준함의 가치를 믿고 있었기에 군대에 있는 18개월 동안에도 개발 관련 일을 계속해서 하고 싶었습니다. 그래서 군대에서도 개발할 수 있는 방법을 찾아보다가 SW개발병이라는 길을 찾았습니다. SW개발병이 되기 위해서는 서류심사와 면접을 통과해야 했습니다. 먼저, 서류를 통과하기 위해서는 컴퓨터 전반에 걸친 지식들을 물어보는 TOPCIT이라는 시험에서 좋은 성적을 거둬야만 했습니다. 그런데 TOPCIT은 기출문제나 모의시험과 같은 정보들이 게시되어 있지 않았고, 학습자료로 올려놓은 것들은 분야별로 나누어진 자습서 뿐이었습니다. 자습서의 내용은 방대한 정보들을 한 책에 담아내고자 축약하고 생략한 내용이 많았습니다. 그렇지만 실제 시험에서도 똑같은 수준의 문제들이 나오는지 알 수 없었습니다. 때문에 자습서의 설명이 부족할 때마다 인터넷을 찾아보거나 집에 있는 책들을 통해 의문점들을 해결해 나갔습니다. 열심히 준비한 끝에 예상한 성적보다 훨씬 좋은 성적을 받아 서류에 통과할 수 있었습니다. 다음으로 면접에서는 저의 실력과 가능성을 보여줘야합니다. 하지만 SW개발병의 업무는 웹 분야가 대부분이었고 당시에는 웹 개발을 단 한 번도 해본 적이 없었습니다. 면접에서 합격하기 위해서는 포트폴리오를 통해 결과물을 보여줘야 한다고 판단했습니다. 웹 개발을 할 때 다양한 분야의 지식이 필요해 난항을 겪었지만 앞서 준비한 TOPCIT에서 배운 지식들이 큰 도움이 되었습니다. 결국 CRUD 프로젝트 하나를 완성해 면접 때 포트폴리오로 가져갔습니다. 그러나 웹 개발에 대한 지식이 모자라 개발 지식을 묻는 질문들에 대답하지 못해 떨어졌고 일반 군으로 입대하게 되었습니다. SW개발병에 떨어지고 난 후 면접을 되돌이켜 보면서 만약 제가 다양한 프로젝트를 진행해보았다면 받았던 다양한 질문들에 대답할 수 있었을 거라는 생각이 들었습니다. 그렇기에 프로젝트 경험을 키우고자 군대를 전역한 이후에는 Node.js를 이용한 성적 관리 웹사이트와 스프링을 이용한 프로젝트를 진행해보았습니다.
